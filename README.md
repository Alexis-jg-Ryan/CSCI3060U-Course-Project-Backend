# CSCI3060U-Course-Project-Backend

## 🎯 Overview

This repository contains the **Back End** component of the Banking System project.

The backend simulates the **overnight batch processing system** used by banks to apply all transactions performed during the day and update the master account records.

Unlike the front end, the backend does **not interact with users directly**. Instead, it processes transaction files generated by the ATM system.

### Responsibilities

The backend system:

- Reads the **Old Master Bank Accounts File**
- Reads the **Merged Bank Account Transaction File**
- Applies all transactions in sequence
- Enforces system constraints (e.g., no negative balances)
- Updates account balances and statuses
- Writes updated output files for the next system run
- Logs errors when constraints are violated

### Output Files

After processing transactions, the backend generates:

- **New Master Bank Accounts File** – updated account records including balances and transaction counts
- **Current Bank Accounts File** – used by the front end for the next session

⚠️ **This is an educational project and NOT production-ready**

## 🚀 Installation

### Prerequisites

- Python 3.8 or higher

### Setup

1. Clone the repository:

```bash
git clone https://github.com/matteogeraldo05/CSCI3060U-Course-Project-Backend.git
cd CSCI3060U-Course-Project-Backend
```

2. Ensure the required input files exist inside the `backend/input` folder:

```
backend/input/
├── old_master_accounts.txt
└── merged_transactions.txt
```

3. Run the backend processor:

```bash
python backend/main.py
```

### File Descriptions

**main.py**  
Entry point for the backend system.

**BackendSystem.py**  
Main backend controller responsible for:
- loading accounts
- processing transactions
- applying business rules
- writing output files

**Transaction.py**  
Defines the transaction structure and parsing logic for transaction records.

**read.py**  
Handles reading and validating the Old Master Bank Accounts File.

**write.py**  
Handles writing the Current Bank Accounts File.

**print_error.py**  
Provides standardized error logging functions.

**input/**  
Contains input files used for processing.

**output/**  
Contains files generated by the backend after processing transactions.

## ⚙️ Backend Processing Flow

```
Old Master Accounts File
        +
Merged Transactions File
        ↓
   Backend System
        ↓
Apply Transactions
        ↓
Update Account Data
        ↓
Write Output Files
```

The backend processes transactions such as:

- Withdrawal
- Transfer
- Paybill
- Deposit
- Create account
- Delete account
- Disable account
- Change account plan

All constraints are validated during processing, and any errors are printed to the terminal.
